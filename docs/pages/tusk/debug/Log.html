    package tusk.debug

    class

    Log

    Macros and functions responsible for logging. To enable or disable specific logs, change the compile-time definitions.

    The following definitions are valid and correspond to the macros defined below:

    • log_fatal
    • log_error
    • log_warning
    • log_info
    • debug
    • log_trace

    Static Methods

    debug ( value : Dynamic ) : Dynamic static

    Use for logging behind-the-scenes information.

    Parameters

    value

    the message / object to log

    error ( value : Dynamic ) : Dynamic static

    Use for crashes and exceptions, and any other errors that aren't necessarily show-stopping but require intervention.

    Parameters

    value

    the message / object to log

    fatal ( value : Dynamic ) : Dynamic static

    Use for show-stopping errors

    Parameters

    value

    the message / object to log

    info ( value : Dynamic ) : Dynamic static

    Use to indicate normal behaviour

    Parameters

    value

    the message / object to log

    log ( v : Dynamic , func : LogFunctions , ? pos : PosInfos ) : Void static

    Replacement for trace, this will format the logs depending on the situation. For example, on web, it will add appropriate colours to each log message in the console.

    Parameters

    v

    the message / object to log

    func

    what type of message it is

    pos

    automatically passed by the compiler

    trace ( value : Dynamic ) : Dynamic static

    Used when delving deep into what is going on

    Parameters

    value

    the message / object to log

    warning ( value : Dynamic ) : Dynamic static

    Use for indicating incorrect behaviour (but execution can be continued).

    Parameters

    value

    the message / object to log
